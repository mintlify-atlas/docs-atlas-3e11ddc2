Agent Skills 是由指令、脚本和资源组成的文件夹，代理可以发现并使用它们来更准确、更高效地完成任务。

## 为什么使用 Agent Skills？

代理的能力越来越强，但往往缺乏可靠完成实际工作所需的上下文。Skills 通过让代理按需加载程序性知识以及公司、团队和用户特定的上下文来解决这个问题。拥有一组技能的代理可以根据正在处理的任务扩展其能力。

## 采用情况

Agent Skills 得到了领先 AI 开发工具的支持，包括 Anthropic Claude、Cursor、VSCode 等平台。

## 开放开发

Agent Skills 格式最初由 [Anthropic](https://www.anthropic.com/) 开发，作为开放标准发布，并已被越来越多的代理产品采用。该标准对来自更广泛生态系统的贡献开放。

[在 GitHub 上查看](https://github.com/devaige/agentskills-cn)

## 主要特性

<AccordionGroup>
  <Accordion title="SKILL.md 格式规范">
    完整的 YAML 前置数据和 Markdown 内容规范，包括 `name`、`description`、`license`、`compatibility` 等字段
  </Accordion>
  <Accordion title="目录结构指南">
    清晰的目录结构指南，包括 `scripts/`、`references/` 和 `assets/` 目录的使用规范
  </Accordion>
  <Accordion title="渐进式披露">
    高效利用上下文的三层结构：元数据（约 100 tokens）、指令（建议 < 5000 tokens）、资源（按需加载）
  </Accordion>
  <Accordion title="验证工具">
    使用 `skills-ref` 库检查技能文件格式和命名规范，确保符合 Agent Skills 规范
  </Accordion>
  <Accordion title="Python CLI 工具">
    提供 `validate`、`read-properties` 和 `to-prompt` 命令，用于验证、读取属性和生成代理提示
  </Accordion>
  <Accordion title="提示生成">
    自动为代理生成标准的 `<available_skills>` XML 格式，便于集成到代理系统提示中
  </Accordion>
</AccordionGroup>
